## 2. 権限モデル: master exclusive unpair

### 2-1. 設計方針

「ペア解除(unpair)」は **masterのみが実行できる操作** として権限モデルを整理する。

| 操作 | v6.0 | v6.1 |
|------|------|------|
| pair doc作成 (create_pair) | master | master (変更なし) |
| pair join (joinByCode) | member (member_uid未設定時) | member (変更なし) |
| pair doc更新 (full update) | partners全員 | **masterのみ** |
| pair deactivate (active=false) | partners全員 (BUG) | **masterのみ** (構造的保証) |

### 2-2. 解消するMinor

| Minor | 内容 | 解消方法 |
|-------|------|--------|
| **M1: app→CF E2E未実証** | unpair()→Firestore write→CF発火の完全アプリフロー未実証(Admin SDK代替で実施) | 本設計後の実機E2E Phase 4aで完全経路を実証 |
| **M2: CTL-6 member edge case** | member側unpair時のpair doc ownership確認未実施 | Firestoreでmasterのみactive=false更新可にする構造的解決 |

### 2-3. 既存運用への影響確認

現在の運用: 奥様=master / 殿=member で稼働中。

| シナリオ | 影響 | 判定 |
|---------|------|------|
| 奥様(master)がunpair → active=false | master_uid一致 → rules許可 | ✅ 影響なし |
| 殿(member)がSettings画面開く | ペア解除ボタン非表示 → UI操作不可 | ✅ 影響なし (既存ペア継続) |
| 殿(member)がアプリコード直接呼出し → unpair() | Firestore rules拒否 → PERMISSION_DENIED | ✅ 構造的ブロック |
| 新規join (inviteCode入力時) | member_uid=null条件でupdate許可 | ✅ 影響なし |

## 3. Firestore rules 変更 (Phase 2)

### 3-1. 変更前 (v6.0)

```
match /pairs/{pairId} {
  allow update: if request.auth != null
    && (request.auth.uid in resource.data.partners
      || resource.data.get('member_uid', null) == null);
}
```

**問題**: `request.auth.uid in resource.data.partners` はmasterもmemberも許可するため、
member側クライアントがFirebaseRepository.unpair()を呼ぶとactive=falseへの更新が成功してしまう(M2)。

### 3-2. 変更後 (v6.1)

```
match /pairs/{pairId} {
  // master_uidのみfull update権限
  // member joinは member_uid未設定時のみ許可
  allow update: if request.auth != null && (
    request.auth.uid == resource.data.master_uid
    || resource.data.get('member_uid', null) == null
  );
}
```

**解消**: memberがjoined後にpair docを更新できるのは、
master_uid一致時のみ。active=false(unpair)はmasterのみ実行可能。

### 3-3. join操作の継続動作確認

`joinByCode()` (FirebaseRepository.kt L147):
- 呼出タイミング: member_uidが未設定のpair doc
- 実行内容: member_uid設定 + joined_at設定 + partnersへ追加
- v6.1 rules評価: `resource.data.get('member_uid', null) == null` → **許可** ✅

## 5. SMV 再モデル化 (Phase 5)

### 5-1. papazon_dash_v2.smv の設計方針

v2 SMVモデルはv1.0.1の実装 (CTL-2/3/6修正済) を正確に反映し、
master exclusive unpair権限モデルを組み込む。

**主要変更**:
1. `ROLE` 変数追加 (`{none, master, member}`)
2. EVENT定義から `member_unpair` を除外 → `master_unpair` のみ存在
3. `create_pair` は ROLE=master のみ実行可能
4. `join_pair` は ROLE=member のみ実行可能
5. CTL-6 member edge caseは新CTL-7として明示: `AG(ROLE = member & PAIR_STATE = paired -> PAIR_DOC_ACTIVE = TRUE)` (TRUEであること、つまりmemberはPAIR_DOC_ACTIVEをFALSEにできない)

### 5-2. 期待するCTL検証結果

| CTL | 性質 | v1固定版(fixed) | v2(本版) |
|-----|------|-------------|--------|
| CTL-1 | AG(AUTH=none→ITEMS_LISTENER=FALSE) | TRUE | TRUE |
| CTL-2 | AG(PAIR_STATE=unpaired→ITEMS_LISTENER=FALSE) | TRUE | TRUE |
| CTL-3 | AG(PAIR_STATE=unpaired→ITEMS_SUBCOL_DATA=FALSE) | TRUE | TRUE |
| CTL-4 | AG(!ORPHAN_USER_DOC) | FALSE (既知limitation) | FALSE (既知limitation) |
| CTL-5 | AG(PAIR_STATE=paired→ITEMS_LISTENER=TRUE) | TRUE | TRUE |
| CTL-6 | AG(PAIR_STATE=unpaired→!PAIR_DOC_ACTIVE) | TRUE | TRUE |
| **CTL-7 (新)** | AG(ROLE=member & PAIR_STATE=paired → PAIR_DOC_ACTIVE=TRUE) | n/a | **TRUE** (member edge case解消) |
